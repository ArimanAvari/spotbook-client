# SpotBook Android Client

Android-клиент для курсовой работы «Личный путеводитель».

Приложение написано на Kotlin с Jetpack Compose. Данные хранятся локально в Room, а синхронизация с Ktor backend выполняется вручную через экран синхронизации. Firebase не используется.

## Технологии

- Kotlin
- Jetpack Compose
- Material Design 3
- Navigation Compose
- ViewModel
- Room
- Ktor Client
- Clean Architecture
- SharedPreferences для JWT-токена

## Запуск

Проект можно открыть в Android Studio как обычный Android/Gradle-проект.

Сборка debug APK:

```powershell
.\gradlew.bat assembleDebug
```

По умолчанию клиент обращается к backend по адресу:

```text
http://10.0.2.2:8080
```

Это удобно для Android Emulator. Для реального устройства можно собрать APK с адресом компьютера в локальной сети:

```powershell
.\gradlew.bat assembleDebug -PbackendBaseUrl=http://192.168.1.10:8080
```

Если устройство подключено по USB и backend работает на этом же компьютере, можно использовать:

```powershell
adb reverse tcp:8080 tcp:8080
.\gradlew.bat assembleDebug -PbackendBaseUrl=http://127.0.0.1:8080
```

## Структура

Основной пакет:

```text
app/src/main/java/com/spotbook/personalguide
```

Слои:

- `domain` - модели, интерфейсы репозиториев и use cases;
- `data` - Room, DTO, API service, репозитории, мапперы, хранение токена;
- `presentation` - Compose-экраны, ViewModel, состояние экранов и навигация.

## Локальная база

Room-база:

```text
spotbook_local.db
```

Таблицы:

- `local_place_cards`
- `local_groups`

Карточки и группы сначала сохраняются локально. Новые и изменённые записи получают `syncStatus = NOT_SYNCED`.

## Backend

Авторизация подключена к Ktor backend:

- регистрация по email/password;
- вход по email/password;
- сохранение JWT-токена локально;
- отправка защищённых запросов с `Authorization: Bearer <token>`;
- кнопка выхода очищает локальный токен.

## Экраны

В приложении есть:

- вход;
- регистрация;
- список карточек;
- создание карточки;
- детали карточки;
- редактирование карточки;
- список групп;
- карточки внутри группы;
- синхронизация.

Интерфейс сделан на Compose и поддерживает поворот экрана. Состояние форм хранится во ViewModel, списки остаются прокручиваемыми, а широкие экраны используют адаптивную ширину контента.

## Синхронизация

Экран синхронизации содержит две ручные операции:

- `Экспорт на сервер` - отправляет локальные карточки и группы со статусами `NOT_SYNCED` и `DELETED`;
- `Импорт с сервера` - загружает карточки и группы текущего пользователя с backend и сохраняет их в Room.

Для MVP используется простое правило:

- экспорт обновляет сервер локальными данными;
- импорт обновляет локальную базу серверными данными.
