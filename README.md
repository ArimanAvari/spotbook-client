# SpotBook Android Client

Android-клиент для курсовой работы «Личный путеводитель».

Приложение написано на Kotlin с Jetpack Compose. Данные хранятся локально в Room, а синхронизация с Ktor backend выполняется вручную через экран синхронизации. Firebase не используется.

## Технологии

- Kotlin
- Jetpack Compose
- Material Design 3
- Navigation Compose
- ViewModel
- Room
- Ktor Client
- Coil
- Clean Architecture
- SharedPreferences для JWT-токена

## Запуск

Проект можно открыть в Android Studio как обычный Android/Gradle-проект.

Сборка debug APK:

```powershell
.\gradlew.bat assembleDebug
```

По умолчанию клиент обращается к backend по адресу:

```text
http://10.0.2.2:8080
```

Это удобно для Android Emulator. Для реального устройства можно собрать APK с адресом компьютера в локальной сети:

```powershell
.\gradlew.bat assembleDebug -PbackendBaseUrl=http://192.168.1.10:8080
```

Если устройство подключено по USB и backend работает на этом же компьютере, можно использовать:

```powershell
adb reverse tcp:8080 tcp:8080
.\gradlew.bat assembleDebug -PbackendBaseUrl=http://127.0.0.1:8080
```

## Структура

Основной пакет:

```text
app/src/main/java/com/spotbook/personalguide
```

Слои:

- `domain` - модели, интерфейсы репозиториев и use cases;
- `data` - Room, DTO, API service, репозитории, мапперы, хранение токена;
- `presentation` - Compose-экраны, ViewModel, состояние экранов и навигация.

## Локальная база

Room-база:

```text
spotbook_local.db
```

Таблицы:

- `local_place_cards`
- `local_groups`

Карточки и группы сначала сохраняются локально. Новые и изменённые записи получают `syncStatus = NOT_SYNCED`.

Фотографии, выбранные пользователем, копируются в локальную папку приложения:

```text
filesDir/place_photos
```

В Room хранится путь к локальному файлу или серверный путь после импорта.

## Backend

Авторизация подключена к Ktor backend:

- регистрация по email/password;
- вход по email/password;
- сохранение JWT-токена локально;
- автоматическое восстановление входа при следующем запуске приложения, если JWT ещё действителен;
- отправка защищённых запросов с `Authorization: Bearer <token>`;
- кнопка выхода очищает локальный токен.

После успешного входа или регистрации приложение проверяет текущий аккаунт. Если пользователь сменился, локальная Room-база очищается и загружаются данные нового аккаунта. Если это тот же пользователь, локальные данные сохраняются, а импорт с backend выполняется безопасным слиянием.

## Экраны

В приложении есть:

- вход;
- регистрация;
- список карточек;
- создание карточки;
- детали карточки;
- редактирование карточки;
- список групп;
- карточки внутри группы;
- синхронизация.

Интерфейс сделан на Compose и поддерживает поворот экрана. Состояние форм хранится во ViewModel, списки остаются прокручиваемыми, а широкие экраны используют адаптивную ширину контента.

На экране создания и редактирования карточки можно выбрать фотографию из галереи. Фото показывается в форме, списке и деталях карточки.

## Синхронизация

Экран синхронизации содержит две ручные операции:

- `Экспорт на сервер` - отправляет локальные карточки и группы со статусами `NOT_SYNCED` и `DELETED`;
- если у карточки выбрано локальное фото, после экспорта карточки файл отправляется на backend через multipart-запрос;
- `Импорт с сервера` - загружает карточки и группы текущего пользователя с backend и сохраняет их в Room;
- серверные фотографии при импорте скачиваются обратно в локальную папку приложения, чтобы они продолжали отображаться после переходов между экранами и повторного входа.

Для MVP используется простое правило:

- экспорт обновляет сервер локальными данными;
- после успешного экспорта локальная база заменяется актуальными серверными данными;
- ручной импорт с сервера не затирает локальные `NOT_SYNCED` и `DELETED` записи;
- если локальная запись уже синхронизирована, она обновляется только когда серверный `updatedAt` новее локального;
- после экспорта данные обновляются слиянием, без полной очистки Room, чтобы локальные пути к фотографиям не терялись.

## Демонстрация

Что можно показать:

1. Запуск приложения.
2. Регистрацию пользователя.
3. Вход пользователя.
4. Создание карточки места.
5. Выбор фотографии для карточки.
6. Просмотр карточки с фотографией в списке.
7. Просмотр фотографии в деталях карточки.
8. Редактирование карточки.
9. Изменение статуса `Хочу посетить` / `Посещено`.
10. Создание группы.
11. Добавление карточки в группу.
12. Удаление карточки из группы без удаления из общего списка.
13. Экспорт данных и фотографии на backend.
14. Импорт данных с backend.
15. Проверку, что данные остаются в Room.
16. Поворот экрана без потери введённых данных.
17. Разделение проекта на `data`, `domain`, `presentation`.
